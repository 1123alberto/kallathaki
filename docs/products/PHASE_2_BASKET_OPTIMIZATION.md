# Kallathaki Phase 2
# Basket Optimization Engine

Version: 1.0

---

# Mission

Transform Kallathaki from a supermarket price comparison app into the smartest basket optimization assistant in Greece.

The application should no longer answer:

"What does this product cost?"

Instead it should answer:

"Where should I shop this week?"

---

# Product Philosophy

Users do not care about individual prices.

Users care about:

- their weekly grocery bill
- convenience
- confidence
- saving money
- reducing effort

Every feature in this phase must reduce one of those five pains.

---

# North Star

Every basket should finish with a recommendation.

Never leave users comparing spreadsheets.

Always answer:

"We recommend this option."

---

# Core Experience

User builds basket.

↓

User taps

Optimize Basket

↓

Kallathaki analyzes every supermarket.

↓

Kallathaki recommends the best shopping plan.

---

# Optimization Levels

## Level 1

Single supermarket

Best total price.

Everything in one place.

---

## Level 2

Two supermarkets

Best balance.

Maximum savings without unnecessary travel.

---

## Level 3

Maximum savings

Three supermarkets if worthwhile.

Only recommend if savings justify another stop.

---

# Decision Engine

Optimization should consider:

✓ Basket total

✓ Product availability

✓ Number of supermarket visits

✓ Savings

✓ Missing products

Future:

Distance

Fuel

Travel time

---

# Recommendation Logic

Never simply display the cheapest result.

Explain WHY.

Examples:

"We recommend Lidl + AB because you'll save €8 while visiting only two stores."

"For only €1.20 more you can buy everything from one supermarket."

---

# User Experience

The optimization process should feel intelligent.

Instead of:

Loading...

Display:

Finding prices...

Checking offers...

Comparing supermarkets...

Building your cheapest basket...

Almost done...
